# 基于 GitHub Pages 的个性化博客网站部署与开发方案

本项目方案旨在为您提供一个从零开始构建、高度可定制样式，且完全支持 `.md` 和 `.kmd` 文件展示的个人博客网站，并最终部署到 GitHub Pages。

## 1. 技术栈选择

为了满足“完全自己设计样式”和“支持自定义扩展名（如 `.kmd`，以及标准 `.md`）”的需求，推荐以下技术组合：
* **构建工具**：[Vite](https://vitejs.dev/) - 极速的前端构建工具。
* **前端框架**：必须与您的笔记应用保持一致：**Vite + Vue 3**。
* **文件解析核心复用（核心）**：
  * 为了实现**完全一致的渲染与样式**，我们需要直接从您的桌面笔记软件中提取代码。
  * 将桌面应用工程中的 `src/renderer/src/utils/markdown-renderer.ts`、`markdown-plugin.ts` 及其相关核心依赖（如 `markdown-it`, `highlight.js`, `katex`, `markdown-it-task-lists` 等）整体迁移至博客前端的 `src/utils/` 目录下。
  * **`.md`与`.kmd` 解析**：直接复用上述模块中的 `renderMarkdownHtml` 函数，无需另外编写解析逻辑，确保渲染器层面的绝对统一。
* **部署平台**：GitHub Pages 结合 GitHub Actions 实现自动化部署。

## 2. 网站目录结构规划

作为网站文件的根目录，`web` 文件夹的推荐结构如下：

```text
web/
├── public/                # 静态资源目录
│   └── posts/             # 建议存放实际的 .md 和 .kmd 博客源文件，方便获取
├── src/
│   ├── assets/            # 网站核心样式文件（CSS/Sass），此处供您完全从零自定义设计
│   │   └── style.css      # 全局主样式，清空默认样式并注入你的个性化风格
│   ├── components/        # 自己设计的 UI 组件（如导航栏、文章列表、文件渲染组件等）
│   ├── utils/
│   │   ├── mdParser.js    # Markdown 渲染与处理辅助函数
│   │   └── kmdParser.js   # 专门针对 .kmd 的转化与渲染逻辑
│   ├── App.vue            # 主页面
│   └── main.js            # 入口文件
├── index.html             # 网站 HTML 模板
├── package.json           # 项目前端依赖
└── vite.config.js         # Vite 配置文件
```

## 3. 核心功能实现方案

### 3.1 动态获取与渲染文件
作为静态博客网站，最好将所有博客文章放入 `public` 文件夹下。在前端，通过 Fetch API 以原文本的方式请求这些内容即可，避免了将大量文件打包进 JS 中：

```javascript
// 示例：动态获取博客原文
async function fetchPost(filename) {
  const response = await fetch(`/posts/${filename}`);
  if (!response.ok) throw new Error('文章加载失败');
  return await response.text();
}
```

### 3.2 提取并装配同源渲染引擎
由于需要与笔记工具 100% 保持一致，请不要重新配置 `markdown-it`，而是进行**代码级复用**：

1. **安装相同的依赖**：从笔记软件的核心 `package.json` 中找到 Markdown 解析相关的依赖，并全部安装在博客项目中：
  ```bash
  npm install markdown-it markdown-it-emoji markdown-it-katex markdown-it-mark markdown-it-task-lists markdown-it-texmath katex highlight.js
  ```
2. **迁移渲染脚本**：将笔记软件源码中的 `src/renderer/src/utils/markdown-renderer.ts` 与 `markdown-plugin.ts` 原封不动（少量剔除了桌面端专属的基础 API 的话进行简单 Mock 即可）地拷贝到博客项目的 `src/utils/` 下。
3. **调用同款渲染函数**：
  ```javascript
  import { renderMarkdownHtml } from './utils/markdown-renderer.ts';

  // 获取到 md/kmd 原文 mdText 之后：
  const htmlContent = renderMarkdownHtml(mdText, null); 
  // 在 Vue 组件中，通过 <div v-html="htmlContent" class="markdown-body"></div> 进行展示
  ```

### 3.3 样式一比一还原的实现思路
您的笔记工具渲染模块（如 `markdown-renderer.ts` 中硬编码的 `exportStyles`）已经包含了所有专门针对表格、代码高亮（GitHub主题等）、Callout (警示块)、Katex 公式的精调样式。
让博客显得同源同宗，您只需：

1. **剥离出核心 CSS **：在前端项目的 `src/assets/` 下创建 `markdown-style.css`。
2. 将原桌面应用 `exportStyles` 字符串内的 CSS 样式全部粘贴进去（或者是原项目现成的全局 markdown-body 样式），并稍作响应式调整（如加宽 `max-width` 限制等）。
3. 在 `main.js` 入口中 `import './assets/markdown-style.css'`。

这样一来，不管是 `.md` 还是 `.kmd` 文件，展现出的基础字号、背景色调、Callout 结构以及任务列表 (`task-list-item`) 等诸多细节上的美学表现，将与桌面版的阅读体验彻头彻尾地拉齐。

## 4. 网站前端美术设计与功能规划

### 4.1 整体设计风格：简约且不失趣味的现代博客风
* **色彩美学**：基础色调采用极简的黑/白/灰（例如 Slate 或 Zinc 色系），但辅以动态、高饱和度的高级渐变色（比如流动的极光色、或者随鼠标移动而跟随的毛玻璃光晕）作为点缀。
* **排版字体**：标题推荐使用无衬线现代字体（如 `Inter`, `Outfit` 或 `Space Grotesk`），结合笔记工具原生的正文字体，营造科技感与呼吸感。
* **微交互特效**：运用 CSS 3D 动画与动画支持库（如 Vue 里的 `@vueuse/motion`）。例如：卡片鼠标悬浮时的轻微 3D 倾斜（磁性效果）、按钮光泽扫过、以及路由切换时极具质感的平滑过渡。
* **完美暗黑模式**：不仅是黑底白字，更是深邃海军蓝或纯黑背景搭配文字、边框的微光效（Neon Glimmer），适应程序员阅读习惯。

### 4.2 核心页面架构与特色玩法设计

根据需求，博客将划分为以下 5 个核心页面：

#### 1. 首页 (Home) - 极具极客范的第一门面
* **美术特征**：首屏引入一个带有动态粒子交互或打字机效果的欢迎区（Hero Section），极简空灵。
* **功能设计**：
  * **瀑布流 / 卡片网格**：以精致的毛玻璃卡片展现最新发布的文章笔记预览（提取前 100 字摘要）。
  * **阅读进度记忆**：前端利用 `localStorage` 记录用户上次看过的文章并提示“继续阅读”。

#### 2. 分类 (Categories) - 秩序井然的知识书架
* **美术特征**：脱离传统列表，将分类设计成拟物化的“文件夹卡片”或带有渐变边框的发光模块。
* **功能设计**：
  * 一对一映射笔记库的目录层级。
  * 统计并在卡片展示该门类下包含多少篇 `.md`/`.kmd` 笔记。点击后平滑展开子目录文章列表。

#### 3. 标签 (Tags) - 动态交互的知识星系
* **美术特征**：打破枯燥罗列。构建一个基于物理引擎（或纯 CSS ）的“标签星系”（Tag Cloud），当鼠标划过，相关标签会连贯发光并轻微放大、重心转移。
* **功能设计**：
  * 解析全部笔记首部的 Metadata 描述来聚合全局标签。
  * 标签热度系统：出现次数越多的标签占地越大、光效越亮。
  * 动态筛选文章：点击标签瞬间在当前页面使用过渡动画（View Transitions API）过滤并显示指定文章。

#### 4. 归档 (Archive) - 重走来时路的时间轴
* **美术特征**：设计一条带有“呼吸灯”和“微光轨迹”效果的垂直折线时间轴。向下滚动页面时，过去的年份和月份节点会被顺次点亮。
* **功能设计**：
  * 按年份、月份维度严格索引所有发表过的笔记。
  * **全维检索增强**：在页面顶部提供具有磨砂质感的强大搜索框，在静态打包后依然支持笔记标题与摘要的快速正则过滤。

#### 5. 个人简介 (About) - 双栏分块的自我展示
* **美术特征**：经典的非对称双栏布局（Split Layout）。文字自述居左，留白充足；右侧设置一个强交互的趣味组件（例如基于 Three.js 渲染的可随鼠标旋转的 3D 键盘/星球模型），或者是一面基于 GitHub Contributions 风格的动态“技能点经验网格墙”。
* **功能设计**：
  * 通过请求一篇特定的 `about.md` 读取博主的自述信息进行渲染。
  * 优雅集成包括 GitHub、推特等多方开源项目与社交媒体的悬浮展示卡片。

## 5. GitHub Pages 自动化部署方案 (GitHub Actions)

借助 GitHub Actions，你可以做到：将本地的笔记内容通过 Git Push 提交后，云端自动编译 Vite 项目并将编译好的网站产物推送到 `gh-pages` 分支进行托管展示。

### 第一步：配置 GitHub Actions 工作流文件
**注意**：此文件需建于您的**整个项目根目录**，即建立 `f:\计算机\develop\noteTool\.github\workflows\deploy-blog.yml`。

内容范本如下：

```yaml
name: Deploy Web Blog to GitHub Pages

on:
  push:
    branches:
      - main  # 假定你的主分支叫 main，如果是 master 请改称 master
    paths:
      - 'web/**'  # 这个策略表示仅当 web 目录里的文件有更改时才触发构建部署

permissions:
  contents: write

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout 🛎️
        uses: actions/checkout@v3

      - name: Setup Node.js ⚙️
        uses: actions/setup-node@v3
        with:
          node-version: '18'

      - name: Install and Build 🔧
        working-directory: ./web     # 指定由于我们项目在 web 文件里，所以工作目录切到 web
        run: |
          npm install
          npm run build

      - name: Deploy 🚀
        uses: JamesIves/github-pages-deploy-action@v4
        with:
          folder: web/dist           # Vite 默认构建出的产物目录
          branch: gh-pages           # 自动推送到负责展示网页的 gh-pages 分支
```

### 第二步：GitHub Pages 设置
1. 在网页端进入你的 GitHub 仓库。
2. 点击上方的 **Settings** 选项卡。
3. 在左侧面板寻找并点击 **Pages**。
4. 在 **Build and deployment** 下的 **Source** 选项，请选择 `Deploy from a branch`。
5. 在分支选择处，选择 `gh-pages` (如果没有出这个分支，可以先推送代码触发一次 Action 让系统自动生成)，并将目录选择为 `/(root)`，点击保存。
6. 一段时间后，您的博客就会自动在这个地址展现：`https://<您的Github用户名>.github.io/<仓库名>`
